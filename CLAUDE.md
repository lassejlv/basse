# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Basse is a self-hosted PaaS in the spirit of Coolify: connect servers, deploy apps from Git/Docker sources, and manage them from a dashboard. See `coolify.md` for the product feature inventory, the progress checklist, and the next implementation targets — it is the source of truth for what to build next.

## Commands

Runtime is **Bun** (not Node). All commands run from the repo root.

- `bun run dev` — run every workspace's `dev` script in parallel (API + web).
- `bun run dev:api` / `bun run dev:web` — run one app. API on `:3000`, web (Vite) on `:5173`.
- `bun run check` — typecheck all workspaces (`tsc --noEmit`, and `tsr generate` + `tsc -b` for web). This is the primary validation gate.
- `bun run lint` — `oxlint`.
- `bun run format` — `oxfmt --write`.
- `bun run db:generate` — generate a Drizzle migration from `packages/db/src/schema.ts`.
- `bun run db:migrate` — apply migrations.

There is no test suite yet. After changes, validate with `bun run check` and `bun run lint`.

Env lives in `.env` at the repo root (copy from `.env.example`); `dev` scripts load it via `--env-file=../../.env`. Vars are read through `Bun.env.*`.

## Architecture

Bun-workspace monorepo: `apps/*` + `packages/*`, isolated linker (`bunfig.toml`).

- **`apps/api`** — Hono server, default export is a Bun server object (`{ port, hostname, fetch }`), run with `bun --hot`. Better Auth is mounted at `/api/auth/*` via `auth.handler`. CORS and Better Auth `trustedOrigins` both allow the Vite dev origins plus `WEB_ORIGIN`.
- **`apps/web`** — Vite + React, TanStack Router (file-based) + TanStack Query. Vite dev-proxies `/api` and `/health` to `:3000`, so the browser talks to the API same-origin in dev. `@/*` aliases `src/*`.
- **`packages/db`** — owns the database. Drizzle ORM on the `bun-sql` Postgres driver. **`prepare: false`** is set deliberately for Neon pooler compatibility. Exports `db`, the schema, and workspace helpers from `@basse/db` (and `@basse/db/schema`).

### Auth + workspaces (the key cross-cutting flow)

Auth is **Better Auth** with the **organization plugin**; an "organization" _is_ a workspace in product terms. On user signup, a `databaseHooks.user.create.after` hook calls `ensurePersonalWorkspace` (`packages/db/src/workspaces.ts`), which creates a personal organization (slug `personal-<userId>`, metadata `{type:"personal"}`) and adds the user as `owner`. Every user therefore always has at least one workspace.

The web app has **no typed API client for auth** — `apps/web/src/lib/organizations.ts` calls Better Auth's organization endpoints (`/api/auth/organization/*`) directly with `fetch` + `credentials: "include"`. Active-workspace selection lives in the sidebar in `apps/web/src/routes/_authed.tsx`.

Better Auth tables (`packages/db/src/auth-schema.ts`) are **generated by the Better Auth CLI** — don't hand-edit them. Basse's own domain tables (`server`, `project`, `app`, `deployment`) live in `packages/db/src/schema.ts`, which re-exports the auth schema so Drizzle sees one combined schema.

### Routing conventions (web)

File-based routing under `apps/web/src/routes`. `routeTree.gen.ts` is generated (`tsr generate`, also run by `check`/`build`) — never edit it by hand. Protected pages live under the pathless `_authed` route, whose `beforeLoad` checks the session and redirects to `/login` otherwise; the route context exposes `session`. Dashboard pages go under `_authed/`.

### UI

shadcn-style components in `apps/web/src/components/ui` built on **Base UI** (`@base-ui/react`, not Radix) and Tailwind CSS v4. `cn()` lives in `src/lib/utils.ts`. Note the API differs from Radix shadcn — e.g. `Select` uses `SelectButton`/`SelectPopup`/`SelectItem`, and components take a `render={<Link/>}` prop for polymorphism rather than `asChild`.

## Conventions

- Domain tables use `text` ids populated with `crypto.randomUUID()` and explicit `createdAt`/`updatedAt` (not DB defaults). Inserts that may race use `.onConflictDoNothing()`.
- `noUncheckedIndexedAccess` is on — array/index access is `T | undefined`; handle it.
- The API is currently auth + health only; routes for projects/servers/apps/deployments and authenticated middleware are not built yet (see `coolify.md`).
